消除歧义的完全限定语法

2020-06-29

消除歧义的完全限定语法

链接


起因： 为同一种类型，实现不同的trait，这些trait中有同名的方法。同名的函数在调用的时候会产生歧义。

如下

trait Pilot {
    fn fly(&self);
}

trait Wizard {
    fn fly(&self);
}

struct Human;

impl Pilot for Human {
    fn fly(&self) {
        println!("This is your captain speaking.");
    }
}

impl Wizard for Human {
    fn fly(&self) {
        println!("Up!");
    }
}

impl Human {
    fn fly(&self) {
        println!("*waving arms furiously*");
    }
}

上面的代码有三个重名的函数，类型自身的方法，以及 Pilot 和 Wizard 实现的方法。

当使用Human类型的实例调用 fly 方法的时候，默认调用自身的方法。

fn main() {
    let person = Human;
    person.fly();
}

为了调用 Pilot trait 或者 wizard trait，需要显示的语法指定。

fn main() {
    let person = Human;
    Pilot::fly(&person);
    Wizard::fly(&person);
    person.fly();
}

关联函数没有self参数，如下。

trait Animal {
    fn baby_name() -> String;
}

struct Dog;

impl Dog {
    fn baby_name() -> String {
        String::from("Spot")
    }
}

impl Animal for Dog {
    fn baby_name() -> String {
        String::from("puppy")
    }
}

fn main() {
    println!("A baby dog is called a {}", Dog::baby_name());
}

Animal::baby_name是一个关联函数，而不是一个方法。它没有self参数，Rust不知道具体类型。

为了消除歧义，采用如下调用

fn main() {
    println!("A baby dog is called a {}", <Dog as Animal>::baby_name());
}

我们为Rust提供了尖括号内的类型注释，完全限定语法格式如下

<Type as Trait>::function(receiver_if_method, next_arg, ...);

可以在调用函数或方法的任何地方使用完全限定语法。
