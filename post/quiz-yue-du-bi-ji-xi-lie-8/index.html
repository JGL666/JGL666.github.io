macro_rules! m {
    (==>) => { print!("1"); };
    (= = >) => { print!("2"); };
    (== >) => { print!("3"); };
    (= =>) => { print!("4"); };
}

fn main() {
    m!(==>);
    m!(= = >);
    m!(== >);
    m!(= =>);
}

解析

NO.1

 (==>) => { print!("1"); };

这个符号Rust中不存在，按照贪心算法，== 被分为一组，>被分为一组

m!(==>);
m!(== >);

这两个都会分组为（==） 和 (>)

NO.2

m!(= = >);

没什么好说的，三个组分别是(=)、(=)、(>)

NO3

(= =>) => { print!("4"); };

没什么好说的，三个组分别是=)、(=>)

Rust相关知识

标点符号在macro_rules宏中会根据Rust中标点符号分组。

=<<

Rust的宏将=<<分组，分组依据是Rust中标点符号，根据贪心算法先取=, 后面的<<，遇到<是一个符号，<<也是一个符号，就将<< 作为一个符号分组
